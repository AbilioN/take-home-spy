# Admin Web App

React (Vite + TypeScript) admin dashboard to authenticate against the Nest backend and inspect user position history.

## Docker usage

When running via `docker compose up --build` from the repository root, the backend serves the built admin UI at:

- `http://localhost:3000/dashboard`

## Production static server (Node, separate port)

After a local build with default `VITE_BASE=/` (do **not** set `VITE_BASE=/dashboard/` for this mode):

```bash
npm run build
npm run start
```

Default URL: **http://localhost:3001** (`ADMIN_STATIC_PORT` overrides the port).

One-shot:

```bash
npm run start:prod
```

## Setup

1. Install dependencies:

```bash
npm install
```

2. Configure API URL:

```bash
cp .env.example .env
```

3. Run:

```bash
npm run dev
```

By default, the app points to `http://localhost:3000`.

## Main flow

- Login with admin credentials (`POST /admin/login`).
- Token is stored in `localStorage` and sent as `x-admin-token`.
- Load users (`GET /admin/users`).
- Select a user and view trajectory history (`GET /admin/users/:id/trajectory`).
# React + TypeScript + Vite

This template provides a minimal setup to get React working in Vite with HMR and some ESLint rules.

Currently, two official plugins are available:

- [@vitejs/plugin-react](https://github.com/vitejs/vite-plugin-react/blob/main/packages/plugin-react) uses [Oxc](https://oxc.rs)
- [@vitejs/plugin-react-swc](https://github.com/vitejs/vite-plugin-react/blob/main/packages/plugin-react-swc) uses [SWC](https://swc.rs/)

## React Compiler

The React Compiler is not enabled on this template because of its impact on dev & build performances. To add it, see [this documentation](https://react.dev/learn/react-compiler/installation).
